        var: 
            i)The scope of a var variable is functional scope.
            ii)It can be updated and re-declared into the scope.
            iii)It can be declared without initialization.
            iv)It can be accessed without initialization as its default value is “undefined”.

        let:
            i)The scope of a let variable is block scope.
            ii)It can be updated but cannot be re-declared into the scope.
            iii)It can be declared without initialization.
            iv)It cannot be accessed without initialization otherwise it will give ‘referenceError’.

        const:
            i)The scope of a const variable is block scope.
            ii)It cannot be updated or re-declared into the scope.
            iii)It cannot be declared without initialization.
            iv)It cannot be accessed without initialization, as it cannot be declared without
            initialization.

                  

        Syntax: A programmer can get the same result as regular functions by writing
         a few lines of code using arrow functions
        This Keyword: Inside of a regular JavaScript function, this value is dynamic.
         The dynamic context means that the value of this depends on how the function is invoked.The
        value of this inside an arrow function remains the same throughout the lifecycle of the
        function and is always bound to the value of this in the closest non-arrow parent function
        which means No matter how or where being executed, this value inside of an arrow function
        always equals this value from the outer function.
        new Keyword: Regular functions are constructible and callable. They can be 
        called using the new keyword.But, the arrow functions are only callable and not constructible,
        i.e., arrow functions can never be used as constructor functions.
